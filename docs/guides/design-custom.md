# Step 3. 디자인 커스텀

> "야, 이거 바꿔줘"라고 AI에게 말하면 됩니다.

---

## 뭘 하는 건가요?

블로그의 **디자인을 내 취향에 맞게 바꾸는** 단계입니다.

지금 상태는 아파트 콘크리트만 올라간 것과 같아요.\
벽지도 없고, 조명도 없고, 간판도 없습니다.\
오늘 **인테리어**를 합니다!

근데 인테리어 업자가 누구냐?\
**AI(Claude)입니다.** 한국어로 원하는 걸 말하면, AI가 코드를 짜줍니다.

---

## AI한테 시키는 방법 — 핵심 원칙

**딱 하나만 기억하세요:**

> **"원하는 걸 한국어로 말한다."** 끝.

### 잘못된 방법 ❌

```
header 컴포넌트의 backgroundColor 속성을 #2563eb로 변경하고
CSS transition을 0.3s ease로 적용해주세요
```

이렇게 전문 용어로 말할 필요 없습니다!

### 올바른 방법 ✅

```
헤더 색깔을 파란색으로 바꿔줘. 부드럽게 전환되게.
```

이렇게 **일상 언어**로 말하면 됩니다.\
Claude가 알아서 코드로 바꿔줍니다.

마음에 안 들면? **"아니, 좀 더 진한 파란색으로"** 하면 됩니다.\
대화하듯이 수정합니다. 정말 쉽죠?

### 💡 팁: 구체적으로 말할수록 결과가 좋아요

| 모호한 요청 ❌ | 구체적인 요청 ✅ |
|--------------|----------------|
| "색 바꿔줘" | "헤더 배경을 진한 남색, 텍스트는 흰색으로 바꿔줘" |
| "예쁘게 해줘" | "글 목록을 카드형으로 바꾸고, 각 카드에 썸네일을 넣어줘" |
| "버튼 추가해" | "헤더 오른쪽에 해/달 아이콘 다크모드 토글 버튼 추가해줘" |

---

## 자주 쓰는 커스텀 — 복사해서 쓰세요!

### 🎨 색상 변경

```
메인 컬러를 #2563eb(파란색)으로 바꿔줘.
헤더, 링크, 버튼 색상 전부 통일.
```

> 💡 색상 코드를 모르겠다고요?\
> "따뜻한 느낌의 주황색으로 바꿔줘" 이렇게 말해도 됩니다!\
> Claude가 적절한 색상을 골라줍니다.

### 📝 사이트 이름(로고) 변경

```
사이트 상단 로고를 "나의 테크 블로그"로 바꿔줘.
폰트는 bold, 크기는 24px로.
```

### 🌙 다크모드 추가

다크모드는 요즘 필수죠! 어두운 곳에서 눈이 편합니다.

```
다크모드 토글 버튼을 헤더 우측에 추가해줘.
- 라이트 모드: 배경 흰색, 텍스트 검정
- 다크 모드: 배경 #1a1a2e, 텍스트 #e0e0e0
- 아이콘은 해/달 모양
- 사용자 설정을 저장해서 다음에 방문해도 유지
```

### 📱 카드형 레이아웃

글 목록을 카드 형태로 바꾸면 더 예쁩니다.

```
블로그 글 목록을 카드형 레이아웃으로 바꿔줘.
- 데스크탑: 3열 그리드
- 태블릿: 2열
- 모바일: 1열
- 각 카드에 썸네일, 제목, 날짜, 요약 표시
```

### 📊 사이드바 추가

```
블로그에 사이드바를 추가해줘.
- 카테고리 목록
- 인기 글 TOP 5
- 간단한 자기소개
- 데스크탑에서만 보이고 모바일에서는 숨김
```

---

## ⚠️ 커스텀할 때 주의할 점

자유도가 높다 보니, 실수하면 **Lighthouse 400점을 깎아먹을 수 있어요!**

### 하지 마세요 ❌

| 실수 | 왜 안 되는가 | 대신 이렇게 |
|------|-------------|-----------|
| 용량 큰 이미지 업로드 (10MB 등) | 사이트가 느려짐 → 성능 점수 하락 | WebP 형식, 500KB 이하 사용 (템플릿이 자동 변환) |
| 외부 폰트 3~4개 로드 | 로딩 시간 증가 | 2개 이하 권장 |
| 과도한 애니메이션 | 화면 떨림(CLS) 발생 | 심플하게, 필수만 |
| 외부 스크립트 많이 추가 | 성능 나락 | Google Analytics 하나면 충분 |

### 안전하게 커스텀하는 프롬프트

불안하면 이 문구를 항상 추가하세요:

```
아래 수정을 해줘.
단, Lighthouse 400점에 영향을 주지 않도록
성능, 접근성, SEO를 유지하면서 작업해줘.

[수정 요청 내용]
```

### 커스텀 후 반드시 확인!

수정이 끝나면 **반드시 Lighthouse를 다시 측정**하세요.

1. [pagespeed.web.dev](https://pagespeed.web.dev)에서 내 사이트 URL 입력
2. 점수가 여전히 400점 근처인지 확인
3. 점수가 많이 떨어졌다면, 방금 한 수정을 되돌리세요

---

## 수정 결과 확인하는 방법

### 1. 내 컴퓨터에서 먼저 확인

```bash
npm run dev
```

`localhost:3000`에서 변경 사항을 확인합니다.

### 2. 반응형 체크 (스마트폰에서도 잘 보이는지)

크롬 브라우저에서:
1. **F12** 누르기 (Mac: `Cmd + Option + I`)
2. 왼쪽 위 📱 아이콘 클릭
3. 다양한 화면 크기에서 확인:
   - 📱 iPhone SE (375px) — 작은 스마트폰
   - 📱 iPad (768px) — 태블릿
   - 💻 Desktop (1280px) — 컴퓨터

### 3. 다크모드 체크

밝은 모드 / 어두운 모드 전환해서 둘 다 깨지지 않는지 확인합니다.
