# Step 3. 디자인 커스텀

> "야, 이거 바꿔줘"라고 AI에게 말하면 됩니다.

---

## Claude Code로 디자인 수정하기

코드를 직접 만질 필요 없습니다.\
Claude Code에게 자연어로 요청하면 코드가 나옵니다.

```
Me (비개발자):  "헤더 우측 상단에 다크모드 토글 버튼 좀 만들어줘.
                아이콘은 달이랑 해 모양으로 하고,
                누르면 부드럽게 전환되게 해줘."

Claude Code:    완성된 DarkModeToggle.tsx 코드 생성 ✅
```

> 복잡한 문법 몰라도 OK.\
> 클린 코드 자동 생성.

---

## 자주 쓰는 커스텀 프롬프트

### 🎨 색상 변경

```
메인 컬러를 #1a1a2e에서 #2563eb로 바꿔줘.
헤더, 링크, 버튼 색상 전부 통일.
```

### 📝 로고 & 사이트명

```
사이트 상단 로고를 "내 블로그 이름"으로 바꿔줘.
폰트는 bold, 크기는 24px로.
```

### 🌙 다크모드

```
다크모드 토글 버튼을 헤더 우측에 추가해줘.
- 라이트: 배경 #ffffff, 텍스트 #333
- 다크: 배경 #1a1a2e, 텍스트 #e0e0e0
- 사용자 설정을 localStorage에 저장
```

### 📱 레이아웃 변경

```
블로그 글 목록을 카드형 레이아웃으로 바꿔줘.
- 데스크탑: 3열 그리드
- 태블릿: 2열
- 모바일: 1열
- 각 카드에 썸네일, 제목, 날짜, 요약 표시
```

### 📊 사이드바 추가

```
블로그에 사이드바를 추가해줘.
- 카테고리 목록
- 인기 글 TOP 5
- 간단한 자기소개
- 데스크탑에서만 보이고 모바일에서는 숨김
```

## 커스텀 시 주의사항

:::warning
**Lighthouse 점수를 깎는 실수들:**

| ❌ 하지 마세요 | ✅ 대신 이렇게 |
|--------------|-------------|
| 구글 폰트 외부 로딩 | 시스템 폰트 스택 사용 |
| 큰 이미지 그대로 사용 | WebP 변환 + lazy loading |
| 과도한 애니메이션 | 필수 애니메이션만 최소로 |
| 외부 라이브러리 남용 | 내장 기능 우선 사용 |
:::

```
성능, 접근성, SEO를 유지하면서 작업해줘.

```

## 디자인 확인하기

수정 후 반드시 3가지를 확인:

### 1. 반응형 체크

```
Chrome → F12 → 디바이스 모드
- iPhone SE (375px)
- iPad (768px)
- Desktop (1280px)
```

### 2. 다크모드 체크

라이트 / 다크 모드 전환해서 둘 다 깨지지 않는지 확인.

### 3. Lighthouse 재측정

```
Chrome → F12 → Lighthouse → Analyze
→ 400점 유지되는지 확인!
```
