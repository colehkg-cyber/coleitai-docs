# Step 4. 테크니컬 SEO 적용

> 템플릿이 대부분 해주지만, 구글에 "내 건물 여기 있어요!" 알려주는 건 직접 해야 합니다.

---

## 1. 구글 서치 콘솔 등록

> 구글에게 내 사이트의 존재를 알리는 단계.

### 등록 방법

1. [search.google.com/search-console](https://search.google.com/search-console) 접속
2. **"속성 추가"** 클릭
3. **"URL 접두어"** 선택 → 내 사이트 주소 입력
4. 소유권 확인 (HTML 파일 업로드 방식 추천)

### 소유권 확인 방법

```
1. 구글이 제공하는 HTML 파일 다운로드
2. 내 프로젝트 /public 폴더에 넣기
3. Github에 커밋 → Vercel 자동 배포
4. 서치 콘솔에서 "확인" 클릭
```

:::info
Claude Code에게 시킬 수도 있습니다:\
`"구글 서치 콘솔 인증 파일 googleXXXX.html을 public 폴더에 추가해줘"`
:::

---

## 2. 사이트맵 제출

> 내 사이트의 전체 지도를 구글에 제출.

1. 서치 콘솔 → 사이드바 → **"Sitemaps"**
2. URL 입력: `sitemap.xml`
3. **"제출"** 클릭

```
상태가 "성공"으로 바뀌면 완료! ✅
```

> 💡 우리 템플릿은 글을 추가할 때마다 sitemap.xml을 자동으로 업데이트합니다.

---

## 3. robots.txt 확인

우리 템플릿에 이미 포함되어 있습니다.\
혹시 수정이 필요하면:

```
# /public/robots.txt

User-agent: *
Allow: /

Sitemap: https://내도메인.com/sitemap.xml
```

---

## 4. 메타 태그 설정

### 글로벌 메타 태그 (사이트 전체)

우리 템플릿의 설정 파일에서 수정:

```
사이트 제목: "내 블로그 이름"
사이트 설명: "이 블로그에 대한 한 줄 설명"
대표 이미지: "/og-image.png" (SNS 공유 시 보이는 이미지)
```

### 글별 메타 태그 (자동)

글을 작성하면 템플릿이 자동으로 생성:
- `<title>` → 글 제목
- `<meta description>` → 글 요약
- Open Graph → SNS 공유용 정보
- Schema.org → 구글 리치 스니펫용

---

## 5. Core Web Vitals 확인

### 서치 콘솔에서 확인

1. 서치 콘솔 → **"Core Web Vitals"**
2. 모바일 / 데스크탑 탭 확인
3. 모든 항목이 **"양호"** 상태인지 확인

### 문제가 있다면

| 문제 | 원인 | 해결 |
|------|------|------|
| LCP 느림 | 이미지가 너무 큼 | 이미지 최적화 (WebP, 압축) |
| CLS 발생 | 이미지 크기 미지정 | width, height 속성 추가 |
| FID 느림 | JavaScript 과다 | 불필요한 스크립트 제거 |

> 💡 우리 템플릿을 수정 없이 사용하면 이 문제들이 거의 발생하지 않습니다.

## 6. (선택) 커스텀 도메인 연결

### Vercel에서 도메인 연결

1. Vercel 대시보드 → 내 프로젝트 → **Settings** → **Domains**
2. 구매한 도메인 입력
3. DNS 설정 안내에 따라 도메인 등록 업체에서 설정

### DNS 설정 (주요 업체)

| 레코드 | 타입 | 값 |
|--------|------|---|
| `@` | A | `76.76.21.21` |
| `www` | CNAME | `cname.vercel-dns.com` |
