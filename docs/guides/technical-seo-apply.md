# Step 4. 테크니컬 SEO 적용

> 건물 다 지었는데, 구글한테 "나 여기 있어요!" 안 알려주면 소용없습니다.

---

## 뭘 하는 건가요?

사이트를 만들었지만, **구글은 아직 모릅니다.**\
"나 여기 있어, 내 사이트 좀 봐줘!" 하고 알려줘야 합니다.

비유하면, 건물을 지었는데 **시청에 건축 신고를 안 한 것**과 같아요.\
신고를 해야 지도에 나오고, 사람들이 찾아올 수 있습니다.

이번 단계에서 하는 일:
1. 구글 서치 콘솔에 내 사이트 등록 (구글에 알리기)
2. 사이트맵 제출 (내 사이트 지도 보내기)
3. 메타 태그 설정 (검색 결과에 어떻게 보일지 정하기)

---

## 1. 구글 서치 콘솔 등록

**구글 서치 콘솔**은 구글이 무료로 제공하는 도구입니다.\
내 사이트가 구글 검색에 어떻게 나오는지 확인하고 관리할 수 있어요.

### 등록 방법 (5분)

1. [search.google.com/search-console](https://search.google.com/search-console)에 접속합니다
2. Google 계정으로 로그인합니다
3. **"속성 추가"** 버튼을 클릭합니다
4. **"URL 접두어"** 방식을 선택합니다
5. 내 사이트 주소를 입력합니다 (예: `https://my-blog.vercel.app`)
6. **소유권 확인**을 합니다 (아래 참고)

### 소유권 확인하기 — "이 사이트가 진짜 내 거예요" 증명

구글이 "정말 당신 사이트 맞아요?" 하고 확인합니다.\
**HTML 파일 방식**이 가장 쉽습니다:

1. 구글이 `googleXXXXXXXX.html` 파일을 줍니다 → **다운로드**합니다
2. 이 파일을 내 프로젝트의 `/public` 폴더에 넣습니다
3. Github에 올립니다:
```bash
git add .
git commit -m "구글 서치 콘솔 인증"
git push
```
4. Vercel이 자동 배포하면 → 서치 콘솔에서 **"확인"** 클릭
5. **"소유권이 확인되었습니다"** 뜨면 성공! ✅

:::info 💡 Claude한테 시킬 수도 있어요!
```
구글 서치 콘솔 인증 파일 googleXXXXXXXX.html을 
public 폴더에 추가해줘
```
파일 이름을 알려주면 Claude가 만들어줍니다.
:::

## 2. 사이트맵 제출

**사이트맵**은 내 사이트의 **지도**입니다.\
"구글아, 나한테 이런 글들이 있어" 하고 목록을 알려주는 거예요.

### 왜 필요한가요?

사이트맵 없이도 구글 봇이 사이트를 찾을 수 있지만,\
**사이트맵이 있으면 훨씬 빨리, 정확하게** 찾아줍니다.

> 💡 우리 템플릿은 글을 추가할 때마다 **사이트맵을 자동 업데이트**합니다.\
> 여러분은 처음 한 번만 제출하면 끝!

### 제출 방법 (1분)

1. 구글 서치 콘솔에 로그인합니다
2. 왼쪽 메뉴에서 **"Sitemaps"** (사이트맵)을 클릭합니다
3. URL 입력칸에 `sitemap.xml`을 입력합니다
4. **"제출"** 버튼을 클릭합니다
5. 상태가 **"성공"**으로 바뀌면 완료! ✅

이제 구글 봇이 여러분 사이트를 **정기적으로 방문**합니다.\
새 글을 올리면 구글이 자동으로 알아차립니다.

---

## 3. robots.txt 확인

`robots.txt`는 구글 봇한테\
**"우리 사이트 전부 봐도 돼"** 알려주는 파일입니다.

> 💡 **우리 템플릿에 이미 포함되어 있습니다.** 건드릴 것 없어요!\
> 혹시 확인하고 싶으면 내 사이트 주소 뒤에 `/robots.txt`를 붙여보세요.\
> 예: `https://my-blog.vercel.app/robots.txt`

---

## 4. 메타 태그 설정

**메타 태그**란, 구글 검색 결과에 보이는 **제목과 설명**입니다.

예를 들어, 구글에서 뭔가를 검색하면 이렇게 나오잖아요:

```
맥북 에어 M3 발열 해결법 5가지        ← 이게 제목 (메타 태그)
맥북 에어 M3 발열 문제를 겪고 계신가요?  ← 이게 설명 (메타 태그)
이 글에서 5가지 해결법을 알려드립니다...
```

이게 매력적이면 **클릭률이 확 올라갑니다!**\
같은 순위라도 설명이 끌리는 사이트를 더 많이 클릭하거든요.

### 설정 방법

우리 템플릿의 **설정 파일**에서 아래만 바꾸면 됩니다:

```
사이트 제목: "나의 테크 블로그"              ← 내 블로그 이름
사이트 설명: "AI와 테크 소식을 쉽게 풀어드립니다"  ← 한 줄 소개
대표 이미지: "/og-image.png"               ← SNS에 공유할 때 보이는 이미지
```

> 💡 **글별 메타 태그는 자동**입니다!\
> 글을 쓸 때 제목과 설명을 넣으면, 검색 결과에 자동으로 반영됩니다.

## 5. Core Web Vitals 확인

서치 콘솔에 등록한 후 며칠이 지나면,\
**"Core Web Vitals"** (핵심 성능 지표) 데이터가 나옵니다.

1. 서치 콘솔 왼쪽 메뉴에서 **"Core Web Vitals"** 클릭
2. 모바일 / 데스크탑 탭 확인
3. 모든 항목이 **"양호"** 상태이면 OK! ✅

> 💡 우리 템플릿을 수정 없이 사용하면\
> Core Web Vitals 문제가 **거의 발생하지 않습니다.**

### 혹시 문제가 있다면?

| 문제 | 왜 생기나 | 어떻게 고치나 |
|------|---------|-------------|
| LCP 느림 (화면이 느리게 뜸) | 이미지가 너무 큼 | 이미지를 압축하거나 WebP로 변환 |
| CLS 발생 (화면이 떨림) | 이미지에 크기를 안 정해서 | width, height 값 추가 |
| FID 느림 (반응이 느림) | 불필요한 스크립트가 많음 | 안 쓰는 스크립트 제거 |

## 6. (선택) 나만의 도메인 연결하기

`my-blog.vercel.app` 대신 **`myblog.com`** 같은 주소를 쓰고 싶다면:

### Vercel에서 연결하기

1. Vercel 대시보드 → 내 프로젝트 → **Settings** → **Domains**
2. 구매한 도메인 주소를 입력합니다
3. Vercel이 DNS 설정 방법을 알려줍니다
4. 도메인 구매 사이트에서 DNS 설정을 합니다:

| 레코드 | 타입 | 값 |
|--------|------|---|
| `@` | A | `76.76.21.21` |
| `www` | CNAME | `cname.vercel-dns.com` |

> 💡 DNS 설정이 뭔지 모르겠다면?\
> Claude에게 물어보세요: **"Vercel에 내도메인.com 연결하는 방법 알려줘"**\
> 또는 도메인 업체 고객센터에 문의하면 친절히 알려줍니다.
