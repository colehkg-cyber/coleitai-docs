# Step 6. AI 글쓰기 실전

> AI가 초안을 쓰고, 내가 경험을 얹고, 구글이 띄워줍니다.

---

## 뭘 하는 건가요?

황금 키워드를 찾았으니, 이제 **그 키워드로 글을 쓰는** 단계입니다.

근데 글을 어떻게 쓰냐고요?\
**AI(Claude)한테 시킵니다.**

"에이, AI가 쓴 글이 검색에 잘 되나요?"\
됩니다. 근데 **조건**이 있어요. AI가 쓴 글을 **그대로 올리면 안 됩니다.**\
거기에 **사람 냄새**를 얹어야 합니다.

오늘 그 전체 과정을 알려드립니다.

---

## AI 글쓰기 전체 과정

```
1. 황금 키워드 선정 (Step 5에서 완료) ✅
       ↓
2. AI에게 글 구조 요청 ← 프롬프트 복사-붙여넣기
       ↓
3. AI가 초안 작성 ← AI가 알아서 해줌
       ↓
4. 내가 검수 + 사람 냄새 추가 ← 이게 핵심!
       ↓
5. 블로그에 올림 → Github 커밋 → 자동 배포
       ↓
6. 구글 서치 콘솔에서 색인 요청 ← 24시간 내 검색 노출
```

여러분이 진짜 노력하는 건 **4번**뿐입니다.\
나머지는 AI가 하거나, 자동이에요.

> 💡 글 하나 쓰는 데 **30분~1시간**이면 됩니다.\
> 예전에 블로그 글 하나 쓰려면 3~4시간 걸렸죠?\
> AI 덕분에 4분의 1로 줄어듭니다.

---

## 핵심 프롬프트 — 그대로 복사해서 쓰세요!

아래 프롬프트를 Claude에 **그대로 복사-붙여넣기** 하세요.\
`[키워드]` 부분만 내 키워드로 바꾸면 됩니다.

```
"[키워드]"로 블로그 글을 써줘.

[구조]
- H1: 글 제목 (60자 이내, 키워드 포함)
- 도입부: 독자의 문제점 짚기 + "이 글에서 해결됩니다" 약속 (200자)
- H2 섹션 4~6개 (각 500자 내외)
- 각 섹션에 표, 리스트, 코드블록 중 하나 이상 포함
- FAQ 3~5개 (구조화 데이터용)
- 결론: 핵심 요약 3줄 + CTA

[SEO 규칙]
- 키워드를 H1, 도입부, H2 1개, 결론에 자연스럽게 포함
- 메타 디스크립션용 요약 1줄 (155자 이내) 별도 제공
- 내부 링크 삽입 위치 2곳 표시 ([내부 링크])

[톤앤매너]
- 전문적이지만 친근한 해요체
- 어려운 용어는 괄호 안에 쉬운 설명
- 구체적 숫자와 데이터 포함

[글자 수]
- 2,500~4,000자
```

### 이 프롬프트가 왜 이렇게 생겼나요?

각 항목에 다 이유가 있습니다:

| 항목 | 왜? |
|------|-----|
| **H1 60자 이내** | 구글 검색 결과에서 제목이 잘립니다. 60자 넘으면 "..."으로 표시돼요 |
| **도입부 200자** | 첫 문장에서 "이 글 나한테 필요한 글이네" 느끼게 만들어야 해요 |
| **H2 섹션 4~6개** | 너무 적으면 얕고, 너무 많으면 집중도가 떨어져요 |
| **표, 리스트 포함** | 구글이 이런 구조화된 내용을 좋아합니다. 검색 결과에 직접 노출될 수도! |
| **FAQ** | 구글이 FAQ를 검색 결과에 펼쳐서 보여줌 → 클릭률 상승! |
| **메타 디스크립션 155자** | 구글 검색 결과 설명문. 155자 넘으면 잘려요 |
| **2,500~4,000자** | 너무 짧으면 구글이 "얕은 글"로 판단. 너무 길면 독자가 이탈 |

## 사람 냄새 추가하기 — 이게 진짜 핵심!

AI가 쓴 글은 **너무 깔끔합니다.**\
문법 완벽, 구조 완벽... 근데 뭔가 밋밋해요.

구글도 바보가 아닙니다. AI가 쓴 글인지 감지할 수 있어요.\
하지만 구글이 싫어하는 건 "AI가 쓴 글" 자체가 아니라, **"가치 없는 글"**입니다.

**사람 냄새를 얹으면 → 유일무이한 가치 있는 글이 됩니다!**

### 반드시 추가할 것 5가지

| # | 추가할 것 | 예시 | 왜 중요한가? |
|---|----------|------|------------|
| 1 | **개인 경험** | "제가 직접 써보니까..." | AI는 이걸 못 만들어요. 독보적! |
| 2 | **실패담** | "처음에 이렇게 했다가 망했습니다" | 진짜 해본 사람만 쓸 수 있음 |
| 3 | **의견** | "솔직히 이건 별로라고 생각합니다" | AI는 의견이 없어요 |
| 4 | **스크린샷** | 실제 화면 캡처 이미지 | 신뢰도 상승 |
| 5 | **구어체 표현** | "진짜 이거 대박이에요" | 사람 냄새! |

> ⚠️ **경험이 없는 분야의 글은 쓰지 마세요!**\
> "나 맥북 에어 없는데 맥북 발열 글 써도 되나요?" → 안 됩니다.\
> 없는 경험을 지어내면 독자가 알아챕니다.\
> **본인이 아는 분야**의 키워드를 선택하세요.

### 사람 냄새 추가도 AI한테 도움받기

```
위 글에 다음을 자연스럽게 추가해줘:
1. 개인적인 사용 경험 1~2개 (내가 나중에 수정할 거니까 플레이스홀더로)
2. "솔직히", "개인적으로는" 같은 구어체 표현
3. 한 곳에 짧은 실패담 (플레이스홀더)
4. [여기에 스크린샷 삽입] 표시
```

이러면 AI가 **"여기에 개인 경험을 넣으세요"** 위치를 표시해줍니다.\
그 자리에 **진짜 본인 경험**을 넣으면 끝!

## 글 발행하기

### 글 파일 만들기

글을 마크다운(.md 또는 .mdx) 파일로 저장합니다.\
파일 맨 위에 **메타 정보**를 넣어요:

```markdown
---
title: "맥북 에어 M3 발열 해결법 5가지 (2026년 최신)"
description: "맥북 에어 M3 발열 문제, 이 5가지 방법으로 즉시 해결하세요."
date: "2026-03-13"
category: "맥북"
thumbnail: "/images/macbook-m3.webp"
---

여기에 본문을 씁니다...
```

### Github에 올리기 → 자동 배포

```bash
git add .
git commit -m "새 글: 맥북 에어 M3 발열 해결법"
git push
```

이 3줄이면 끝! Vercel이 자동으로 사이트를 업데이트합니다.

### 구글에 색인 요청하기

글을 올렸으면 구글한테 **"새 글 올렸어, 빨리 와봐!"** 알려줍니다.

1. [구글 서치 콘솔](https://search.google.com/search-console)에 로그인
2. 위쪽 검색창에 **새 글의 URL**을 입력합니다
3. **"색인 생성 요청"** 클릭

> 💡 400점짜리 사이트는 보통 **24시간 이내**에 구글에 뜹니다!\
> 색인 요청 안 해도 며칠 내로 뜨지만, 하면 **더 빨리** 뜹니다.

## 글 발행 주기 — 얼마나 자주 써야 하나요?

| 시기 | 권장 주기 | 이유 |
|------|----------|------|
| 초기 (1~3개월) | **주 2~3개** | 구글 신뢰도를 빨리 쌓아야 해요 |
| 안정기 (4개월~) | **주 1개** | 유지하면서 기존 글 업데이트도 |
| 자동화 단계 | **월 2~3개** | 복리 효과로 글 안 써도 트래픽 유지 |

> 🎯 **핵심: 꾸준함!**\
> 한 달에 10개 몰아쓰고 2개월 쉬는 것보다,\
> **일주일에 1개씩 꾸준히** 쓰는 게 훨씬 낫습니다.\
> 구글은 지속적으로 업데이트되는 사이트를 좋아합니다.

### 글이 쌓이면 어떻게 되나요?

| 글 수 | 예상 월 트래픽 | 비유 |
|-------|-------------|------|
| 10개 | 500~1,000명 | 원룸 건물 |
| 30개 | 3,000~5,000명 | 소형 빌딩 |
| 50개 | 10,000~20,000명 | 중형 빌딩 |
| 100개+ | 50,000명+ | 대형 빌딩 |

## 체류 시간 늘리는 글쓰기 팁

구글은 방문자가 **오래 머무는 글**을 좋아합니다.\
아래 기법을 쓰면 체류 시간이 올라갑니다:

| 기법 | 쉬운 설명 | 예시 |
|------|----------|------|
| **호기심 갭** | 다음 내용이 궁금하게 만들기 | "하지만 진짜 중요한 건 다음입니다" |
| **패턴 인터럽트** | 예상을 깨기 | "대부분 여기서 틀립니다" |
| **구체적 숫자** | 모호하게 쓰지 않기 | "3.2초 만에" (❌ "빠르게") |
| **표/리스트** | 글만 쭉 이어지지 않게 | 3문장 연속하면 표나 리스트 넣기 |
