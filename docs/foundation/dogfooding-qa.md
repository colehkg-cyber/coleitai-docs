# 5강. 개밥먹기 & QA

> **내가 안 쓰는 제품을 남한테 쓰라고?** 직접 써봐야 진짜가 보인다.

---

## 개밥먹기 (Dogfooding)

"Eating your own dog food" — 자기 회사 제품을 자기가 직접 쓰는 것.

| 회사 | 개밥먹기 |
|------|---------|
| Microsoft | 직원 전원 Windows 사용 |
| Google | 직원 전원 Chrome 사용 |
| Slack | 전 직원이 Slack으로 소통 → 불편한 점 직접 발견 |
| Notion | 전 직원이 Notion으로 문서 관리 → 성능 이슈 직접 체감 |

### 콘텐츠 사업에서의 개밥먹기

내 블로그 글을 **"검색자 입장"**에서 직접 읽어보는 것.

체크해야 할 것:
- 📱 **모바일에서 읽어봤나?** (트래픽 70%가 모바일)
- ⏱️ **3분 안에 답을 찾을 수 있나?**
- 📜 **스크롤이 자연스럽나?**
- 🔗 **링크가 다 작동하나?**
- 🖼️ **이미지가 제대로 뜨나?**

> **이 질문에 No가 하나라도 있으면, 독자도 No다.**

## QA (Quality Assurance)

**품질 보증** — 출시 전에 문제를 찾아서 고치는 것.

버그가 유저에게 도달하기 전에 잡아야 합니다. 유저가 버그를 발견하면 = 신뢰 하락 = 이탈.

### QA의 4가지 유형

| 유형 | 질문 | 콘텐츠 적용 |
|------|------|------------|
| **기능 테스트** | 의도대로 작동하는가? | 링크 클릭 → 올바른 페이지로 이동? |
| **디바이스 테스트** | 모든 기기에서 잘 보이나? | PC, 모바일, 태블릿 전부 확인 |
| **성능 테스트** | 빠르게 로딩되나? | Lighthouse 점수 확인 |
| **사용성 테스트** | 유저가 쉽게 쓸 수 있나? | 3클릭 안에 원하는 글 찾을 수 있나? |

## 콘텐츠 QA 체크리스트

글 발행 전 반드시 확인해야 할 8가지:

| # | 항목 | 체크 내용 | 이유 |
|---|------|----------|------|
| 1 | **팩트 체크** | AI 작성 내용 중 숫자/날짜/이름 크로스체크 | 할루시네이션 방지 |
| 2 | **모바일 확인** | 스마트폰에서 직접 읽어보기 | 트래픽 70%가 모바일 |
| 3 | **키워드 확인** | H1, 도입부, H2, 결론에 키워드 포함? | SEO 기본 |
| 4 | **링크 확인** | 내부/외부 링크 전부 클릭해서 확인 | 깨진 링크 = 구글 감점 |
| 5 | **이미지 확인** | alt 태그 있나? 용량 적절? | 접근성 + 성능 |
| 6 | **소리내어 읽기** | 어색한 부분은 AI한테 수정 | 자연스러움 = 품질 |
| 7 | **메타 확인** | 제목 60자, 설명 155자 이내 | 검색 결과 잘림 방지 |
| 8 | **속도 확인** | Lighthouse 돌려서 점수 확인 | 성능 = 순위 = 돈 |

:::warning ⚠️ 규칙
**체크 안 한 글 = 발행 금지.** 1번(팩트체크)은 특히 중요합니다.
AI가 만든 숫자는 반드시 검증하세요.
:::

## 버그 리포트 작성법

문제를 발견하면 — **기록하라.**

### 템플릿

| 항목 | 예시 |
|------|------|
| **제목** | [모바일] 글 제목이 잘림 |
| **환경** | iPhone 15, Safari, iOS 18 |
| **재현 순서** | 1. 블로그 접속 → 2. 글 클릭 → 3. 제목 확인 |
| **기대 결과** | 제목 전체가 보여야 함 |
| **실제 결과** | 제목이 "맥북 배터리..."로 잘림 |
| **심각도** | Medium |
| **스크린샷** | [첨부] |

### 심각도 레벨

| 레벨 | 의미 | 대응 |
|------|------|------|
| 🔴 **Critical** | 사이트 접속 불가, 데이터 유실 | 즉시 수정 |
| 🟠 **High** | 핵심 기능 작동 안 함 | 24시간 내 |
| 🟡 **Medium** | 기능은 되지만 UX 불편 | 이번 스프린트 |
| 🟢 **Low** | 사소한 UI 이슈 | 백로그 |

**콘텐츠 심각도 기준:**
- 잘못된 정보 = 🔴 Critical
- 깨진 링크 = 🟠 High
- 느린 이미지 = 🟡 Medium
- 오탈자 = 🟢 Low

## 엣지 케이스

**"보통"이 아닌 상황을 테스트하라.** 보통은 잘 되는데, 이런 상황에서 터진다.

| 엣지 케이스 | 해결 |
|------------|------|
| 글 제목이 100자면? | 검색 결과에서 잘림 → 60자 이내 규칙 |
| 인터넷 느린 3G에서? | 이미지 로딩 실패 → WebP + lazy loading |
| 다크 모드에서 읽히나? | 텍스트 안 보임 → CSS 변수 체크 |
| 글이 10,000자면? | 스크롤 피로 → 목차(TOC) 필요 |
| 해외에서 접속하면? | CDN 미배포 → Vercel Edge 확인 |

## 베타 테스트 & 피드백

| 단계 | 이름 | 콘텐츠 적용 |
|------|------|------------|
| **Alpha** | 내부 테스트 | 내가 먼저 읽어보기 |
| **Beta** | 외부 제한 공개 | 지인 3명에게 보여주기 |
| **RC** | 출시 후보 | 서치 콘솔 색인 요청 |
| **GA** | 정식 출시 | 구글에 노출 시작! |

:::tip 🎯 최고의 피드백 질문
"좋은데?"가 아니라 **"어디가 불편해?"**를 물어라.
지인 3명에게 보여줬을 때의 반응 = 가장 정직한 피드백.
:::
