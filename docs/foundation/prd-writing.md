# 4강. PRD 작성법

> **PRD(Product Requirements Document) = 아이디어를 문서로.** 문서가 되면 실행력이 10배 올라간다.

---

## PRD가 뭔가?

**PRD = 제품 요구사항 문서** — "뭘 만들 건지"를 구체적으로 적은 문서.

| 비유 | PRD |
|------|-----|
| 건축 설계도 | 뭘 짓는지, 어떤 구조인지, 언제까지인지 |
| 요리 레시피 | 재료 뭐 필요하고, 순서가 어떻게 되는지 |
| 여행 계획서 | 어디 가고, 뭘 하고, 예산이 얼마인지 |

**PRD가 없으면?**
"대충 이런 거 만들어줘" → 서로 다른 걸 상상 → 시간 낭비 → 처음부터 다시

---

## 1페이지 PRD 템플릿

실리콘밸리에서도 처음부터 30페이지 PRD를 쓰지 않습니다. **1페이지로 시작합니다.**

```markdown
# [제품/기능 이름]

## 문제 (Problem)
사용자가 겪고 있는 문제가 무엇인가?

## 목표 (Goal)
이걸 해결하면 어떤 상태가 되는가?

## 성공 지표 (Metrics)
어떻게 측정할 것인가?

## 핵심 기능 (Key Features)
무엇을 만들 것인가? (우선순위 포함)

## 일정 (Timeline)
언제까지 할 것인가?

## 리스크 (Risks)
뭐가 안 될 수 있는가?
```

---

## 콘텐츠 PRD 예시

```markdown
# "맥북 배터리" 클러스터 PRD

## 문제
맥북 사용자들이 배터리 관련 정보를 검색하지만,
한국어 양질의 콘텐츠가 부족하다.

## 목표
"맥북 배터리" 관련 키워드 10개에서
구글 1페이지 (상위 10위) 달성.

## 성공 지표
- 월 트래픽 5,000 이상
- 평균 체류시간 2분 이상
- 애드센스 RPM $10 이상

## 핵심 기능 (글 목록)
P0 (필수):
- 맥북 배터리 오래 쓰는 법 10가지
- 맥북 배터리 교체 비용 총정리
- 맥북 배터리 수명 확인 방법

P1 (중요):
- 맥북 에어 vs 프로 배터리 비교
- 맥북 배터리 보정 방법

P2 (나중에):
- macOS 배터리 설정 완전 가이드
- 맥북 충전기 추천

## 일정
- Week 1: P0 글 3편 발행
- Week 2: P1 글 2편 발행
- Week 3~4: 성과 확인 + 업데이트

## 리스크
- 경쟁이 심한 키워드일 수 있음 → 롱테일 키워드 우선
- AI 글 품질이 낮을 수 있음 → 팩트체크 필수
```

---

## MoSCoW 우선순위

기능(글)의 우선순위를 정하는 프레임워크:

| 등급 | 의미 | 콘텐츠 적용 |
|------|------|------------|
| **Must** | 반드시 있어야 함 | 검색량 높은 핵심 키워드 글 |
| **Should** | 있어야 하지만 필수는 아님 | 관련 키워드 글 |
| **Could** | 있으면 좋음 | 롱테일 키워드 글 |
| **Won't** | 이번엔 안 함 | 경쟁 너무 심한 키워드 |

:::tip 🎯 실전 팁
처음에는 **Must 3개만** 집중하세요.
나머지는 Must가 성과를 내기 시작하면 추가합니다.
:::

## PRD → AI 프롬프트 연결

PRD를 잘 쓰면, AI 프롬프트도 잘 쓸 수 있습니다.

| PRD 항목 | AI 프롬프트 변환 |
|----------|----------------|
| 문제 | "~를 검색하는 사용자를 위한" |
| 목표 | "~를 달성하기 위해" |
| 타겟 | "~를 대상으로" |
| 핵심 기능 | "다음 내용을 포함해서" |
| 성공 지표 | "~에 최적화된" |

### 예시 변환

```
PRD:
  문제: 맥북 배터리가 빨리 닳아서 불편한 사용자
  목표: 실행 가능한 배터리 절약 팁 제공
  타겟: IT에 익숙하지 않은 일반 맥북 사용자

AI 프롬프트:
  "맥북 배터리가 빨리 닳아서 불편한 일반 사용자를 위해,
   IT 지식이 없어도 바로 따라할 수 있는
   배터리 절약 팁 10가지를 작성해주세요.
   각 팁은 3줄 이내로 간결하게,
   구글 SEO에 최적화된 구조로 작성해주세요."
```

## 핵심 정리

| 개념 | 한 줄 |
|------|-------|
| PRD | 뭘 만들 건지 적은 문서 = 실행력 10배 |
| 1페이지 PRD | 문제/목표/지표/기능/일정/리스크 |
| MoSCoW | Must > Should > Could > Won't 우선순위 |
| PRD→프롬프트 | PRD 잘 쓰면 AI 프롬프트도 잘 쓸 수 있다 |
