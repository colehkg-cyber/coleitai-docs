# 2강. API란 무엇인가

> **API = 소프트웨어끼리 대화하는 방법.** 레스토랑 웨이터라고 생각하면 된다.

---

## 레스토랑 비유

```
[당신 = 클라이언트]  →  [웨이터 = API]  →  [주방 = 서버]

1. 메뉴판 보고 주문 (요청 / Request)
2. 웨이터가 주방에 전달 (API 호출)
3. 주방에서 요리 (서버 처리)
4. 웨이터가 음식 가져옴 (응답 / Response)
```

이게 API의 전부입니다.

- **메뉴판** = API 문서 (어떤 요청이 가능한지)
- **주문** = Request (요청)
- **음식** = Response (응답)
- **주방 규칙** = API 규칙 (형식, 인증 등)

---

## REST API — 가장 흔한 형태

REST(Representational State Transfer)는 API의 가장 보편적인 형태입니다.

### HTTP 메서드

| 메서드 | 레스토랑 비유 | 실제 의미 |
|--------|-------------|----------|
| **GET** | "메뉴 보여주세요" | 데이터 조회 |
| **POST** | "이거 주문할게요" | 데이터 생성 |
| **PUT** | "주문 변경할게요" | 데이터 수정 |
| **DELETE** | "주문 취소할게요" | 데이터 삭제 |

### 실제 예시

```
GET  https://api.weather.com/seoul     → 서울 날씨 가져와
POST https://api.blog.com/articles     → 새 글 발행해
PUT  https://api.blog.com/articles/1   → 1번 글 수정해
DELETE https://api.blog.com/articles/1 → 1번 글 삭제해
```

## JSON — API의 언어

API가 주고받는 데이터 형식이 JSON(JavaScript Object Notation)입니다.

```json
{
  "title": "맥북 배터리 오래 쓰는 법",
  "author": "콜잇AI",
  "views": 15000,
  "tags": ["맥북", "배터리", "팁"],
  "published": true
}
```

**규칙:**
- `{ }` 중괄호로 감싸기
- `"키": "값"` 쌍으로 구성
- 문자열은 `" "` 따옴표
- 숫자는 따옴표 없이
- true/false는 따옴표 없이

:::tip 💡 왜 JSON을 알아야 하나?
서치 콘솔 API, 애드센스 API, AI API — 전부 JSON으로 데이터를 주고받습니다.
"이게 뭔지" 알면 에러가 생겨도 당황하지 않습니다.
:::

## API 인증 — 아무나 주방에 못 들어간다

### API Key
```
GET https://api.example.com/data?key=abc123xyz
```
레스토랑 멤버십 카드 같은 것. 요청할 때마다 "나 회원이에요" 증명.

### OAuth 2.0
Google, Facebook 로그인에 쓰이는 방식.
1. "구글 로그인" 클릭
2. 구글이 "이 사람 맞습니다" 토큰 발급
3. 그 토큰으로 API 사용

### Bearer Token
```
Authorization: Bearer eyJhbGciOiJIUzI1...
```
입장 팔찌 같은 것. 한 번 받으면 그걸로 계속 입장.

## API 에러 코드 — 웨이터가 뭐라고 하는지

| 코드 | 의미 | 레스토랑 비유 |
|------|------|-------------|
| **200** | 성공 | "주문하신 음식 나왔습니다" |
| **201** | 생성 성공 | "새 메뉴 등록 완료" |
| **400** | 잘못된 요청 | "그런 메뉴 없는데요..." |
| **401** | 인증 실패 | "회원카드 보여주세요" |
| **403** | 권한 없음 | "VIP 전용입니다" |
| **404** | 못 찾음 | "그 메뉴 품절입니다" |
| **429** | 너무 많은 요청 | "잠시만 기다려주세요, 주문이 밀려서..." |
| **500** | 서버 에러 | "주방에 불났어요!" |

:::tip 🎯 실전 팁
`429`가 뜨면 → 잠시 기다렸다가 다시 시도.
`401`이 뜨면 → API 키 확인.
`500`이 뜨면 → 내 잘못 아님. 서버 문제. 기다리면 됨.
:::

## 핵심 정리

| 개념 | 한 줄 |
|------|-------|
| API | 소프트웨어끼리 대화하는 방법 (웨이터) |
| REST | API의 가장 보편적 형태 |
| JSON | API가 주고받는 데이터 형식 |
| API Key | "나 인증된 사용자야" 증명 |
| 상태 코드 | 200=성공, 4xx=내 잘못, 5xx=서버 잘못 |
