# 1강. 제품개발의 세계관

> **PM과 PO가 뭔지, 애자일과 스프린트가 뭔지 — 이것만 알면 개발팀과 대화가 된다.**

---

## PM vs PO — 뭐가 다른가?

| | PM (Product Manager) | PO (Product Owner) |
|---|---|---|
| **한 줄 요약** | 제품의 CEO | 백로그의 주인 |
| **핵심 질문** | "왜 이걸 만드는가?" | "무엇을 먼저 만드는가?" |
| **시야** | 시장·비전·전략 (Why + What) | 팀·실행·우선순위 (What + How) |
| **주요 업무** | 시장 분석, 로드맵, 비즈니스 전략 | 백로그 관리, 스프린트 계획, 팀 소통 |
| **소통 대상** | CEO, 투자자, 마케팅 | 개발팀, 디자이너, QA |

:::tip 💡 콘텐츠 사업에서는?
여러분이 PM이자 PO입니다.
- PM 역할: "어떤 주제(키워드)로 어떤 방향의 블로그를 만들까?"
- PO 역할: "이번 주에는 어떤 글을 먼저 쓸까?"
:::

## 워터폴 vs 애자일

### 워터폴 (Waterfall) — 전통적 방식

```
요구사항 정의 → 설계 → 개발 → 테스트 → 출시
     1개월       1개월    2개월    1개월    = 5개월 후 출시
```

**문제:** 5개월 후에 내놓았더니 시장이 바뀌어 있다. 혹은 유저가 원하는 게 아니었다.

### 애자일 (Agile) — 실리콘밸리 표준

```
[Sprint 1: 2주] → 출시 → 피드백 →
[Sprint 2: 2주] → 출시 → 피드백 →
[Sprint 3: 2주] → 출시 → 피드백 → ...
```

**핵심:** 작게 만들고, 빨리 내보내고, 피드백으로 개선.

| | 워터폴 | 애자일 |
|---|---|---|
| 계획 | 처음에 전부 | 매 스프린트 |
| 출시 | 한 번에 완성품 | 2주마다 작동하는 버전 |
| 변경 | 어렵고 비쌈 | 매 스프린트 가능 |
| 위험 | 마지막에 폭발 | 매번 조금씩 관리 |

:::tip 🎯 콘텐츠에 적용하면?
- 워터폴: 100편 다 쓰고 한꺼번에 발행 → ❌
- 애자일: 주 2편 발행 → 데이터 확인 → 방향 수정 → ✅
:::

## 스프린트 사이클

스프린트는 보통 **1~2주 단위**로 돌아갑니다.

### 스프린트 구성

| 단계 | 할 일 | 콘텐츠 적용 |
|------|-------|------------|
| **계획 (Planning)** | 이번 스프린트에 뭘 할지 결정 | "이번 주 키워드 3개 선정" |
| **실행 (Execution)** | 매일 조금씩 만든다 | 월~금 하루 1시간 글 작성 |
| **데일리 체크** | 15분 짧은 점검 | 서치 콘솔 5분 체크 |
| **리뷰 (Review)** | 결과물 확인 | 발행한 글 성과 확인 |
| **회고 (Retro)** | 뭐가 잘됐고 뭐가 안됐는지 | "이 키워드는 됐고, 저건 안 됐다" |

### 콘텐츠 스프린트 예시 (2주)

```
Week 1:
  월 - 키워드 리서치 (3개 선정)
  화 - 글 #1 초안 (AI 작성)
  수 - 글 #1 수정 + 발행
  목 - 글 #2 초안 (AI 작성)
  금 - 글 #2 수정 + 발행

Week 2:
  월 - 글 #3 초안 (AI 작성)
  화 - 글 #3 수정 + 발행
  수 - 기존 글 업데이트 (순위 올리기)
  목 - 데이터 분석 (서치 콘솔)
  금 - 회고 + 다음 스프린트 키워드 선정
```

## MVP (Minimum Viable Product)

**최소 기능 제품** — 핵심 기능만 넣어서 시장 반응을 빠르게 확인하는 것.

### 잘못된 MVP vs 올바른 MVP

```
❌ 잘못된 MVP:
바퀴 → 프레임 → 엔진 → (6개월 후) 완성차

✅ 올바른 MVP:
스케이트보드 → 자전거 → 오토바이 → 자동차
(매 단계에서 "이동"이라는 핵심 가치를 제공)
```

:::tip 🎯 콘텐츠 MVP
- ❌ 100편 다 쓰고 완벽한 블로그 오픈
- ✅ 핵심 키워드 5편으로 시작 → 반응 보고 확장

"80점짜리 글 5편 > 100점짜리 글 0편"
:::

## PMF (Product-Market Fit)

**제품-시장 적합성** — 시장이 진짜 원하는 제품을 만들었는가?

### PMF를 찾았다는 신호

| 신호 | 제품 | 콘텐츠 |
|------|------|--------|
| 자연 성장 | 마케팅 없이 유저 증가 | 검색 유입이 알아서 증가 |
| 높은 리텐션 | 유저가 매일 돌아옴 | 독자가 재방문 + 구독 |
| 입소문 | "이거 좋다" 공유 | 글이 SNS에서 공유됨 |
| 불만 대신 요청 | "이 기능 추가해주세요" | "이 주제도 써주세요" |

### PMF를 못 찾았다는 신호

- 트래픽이 0에 수렴
- 체류시간이 30초 미만
- 이탈률 90% 이상
- 검색 순위가 절대 안 올라감

→ **피봇이 필요하다.** 키워드 변경, 주제 변경, 타겟 변경.

## 핵심 정리

| 개념 | 한 줄 | 콘텐츠 적용 |
|------|-------|------------|
| PM/PO | 왜 만드는가 + 뭘 먼저 만드는가 | 전략 + 우선순위를 내가 결정 |
| 애자일 | 작게 빠르게 반복 | 주 2편 발행 → 데이터 → 개선 |
| 스프린트 | 2주 단위 실행 | 콘텐츠 스프린트 운영 |
| MVP | 최소 기능 먼저 | 핵심 키워드 5편으로 시작 |
| PMF | 시장이 원하는가? | 트래픽 + 리텐션으로 확인 |
