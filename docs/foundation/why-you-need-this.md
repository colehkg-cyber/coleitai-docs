# 0강. 왜 이걸 알아야 하는가

> **콘텐츠도 제품이다.** 이 관점이 잡히면 모든 게 달라진다.

---

## 이 강의를 만든 사람

디자이너 출신 → PM → PO → 그로스 PM → 마케터 → 세일즈 → BD

실리콘밸리 스타트업 Turing의 초기 멤버로, 두 가지 제품을 만들었습니다:

| 프로젝트 | 성과 | 역할 |
|----------|------|------|
| **수학대왕** | 누적 200만 사용자 | PM / Growth |
| **GPAI** | 3개월 만에 글로벌 70만 사용자 | PM / PO / Growth |

이 숫자는 **운이 아니라 방법론의 결과**입니다. 그리고 그 방법론은 콘텐츠 사업에도 그대로 적용됩니다.

---

## 블로그 = 제품

대부분의 사람들은 블로그를 "일기장"이나 "취미"로 생각합니다.

하지만 제품개발 관점에서 보면:

| 제품 개발 | 콘텐츠 사업 |
|-----------|-------------|
| 사용자 리서치 | 키워드 리서치 (사람들이 뭘 검색하는가?) |
| MVP 개발 | 글 1개 발행 (최소한의 콘텐츠로 시장 반응 확인) |
| PMF 확인 | 검색 순위 + 트래픽 (시장이 원하는 콘텐츠인가?) |
| 스케일링 | 글 확장 + 수익화 (트래픽 → 광고 → 제휴 → 상품) |

**글 하나 = 미니 제품.** 블로그 전체 = 제품 포트폴리오.

---

## 제품적 사고의 5가지 요소

### 1. 사용자 (User)
누가 이걸 쓰는가? → 누가 이 글을 읽는가?

"맥북 배터리 오래 쓰는 법"을 검색하는 사람이 원하는 건:
- ✅ 지금 당장 쓸 수 있는 팁
- ❌ 배터리의 화학 원리

**사용자를 모르면 아무것도 만들 수 없다.**

### 2. 문제 (Problem)
왜 이걸 만드는가? → 이 글이 어떤 문제를 해결하는가?

검색 = 문제 선언. "맥북 배터리 오래 쓰는 법" = "배터리가 빨리 닳아서 불편하다."

### 3. 해결책 (Solution)
어떻게 해결하는가? → 글의 본문

"10가지 설정 변경으로 배터리 2배 오래 쓰기" — 구체적이고 실행 가능한 해결책.

### 4. 측정 (Metric)
잘 되고 있는지 어떻게 아는가?

- DAU (일일 방문자)
- 체류시간 (글을 얼마나 읽는가)
- CTR (검색 결과에서 클릭률)

### 5. 반복 (Iteration)
어떻게 더 좋게 만드는가?

만들고 → 측정 → 학습 → 반복. 이걸 **린(Lean)** 사이클이라고 합니다.

:::tip 💡 핵심
콘텐츠도 제품이다. 이 관점이 잡히면, 글 쓰는 방식이 완전히 달라집니다.
:::

---

## 제품적 사고 프레임워크

콘텐츠 사업에 제품적 사고를 적용하면 이런 단계를 따릅니다:

| 단계 | 제품 용어 | 콘텐츠 사업 적용 |
|------|----------|-----------------|
| 1. 문제 발견 | 사용자 리서치 | 키워드 리서치 = 시장 조사 |
| 2. 가설 수립 | PRD 작성 | "이 키워드로 이 글을 쓰면 트래픽이 올 것이다" |
| 3. MVP 제작 | 개밥먹기 | 80점짜리 글을 빠르게 발행 → 내가 먼저 읽어보기 |
| 4. 측정 | 데이터 분석 | 서치 콘솔에서 노출/클릭 확인 |
| 5. 반복 | 그로스 | 4~10위 글 업데이트 → 상단 진입 |

---

## 실리콘밸리 DNA

실리콘밸리가 다른 이유는 4가지 원칙 때문입니다:

### 🚀 Ship Fast — 빨리 내보내라
완벽한 걸 6개월 만드는 것보다, 괜찮은 걸 1주에 내보내는 게 낫다.

> "Done is better than perfect." — Mark Zuckerberg

**콘텐츠 적용:** 완벽한 글 1개보다 괜찮은 글 4개가 낫다. 일단 발행하고 나중에 업데이트.

### 📊 Data Driven — 데이터로 말해라
"나는 이게 좋다고 생각해요" → "데이터가 이렇게 말합니다"

감보다 숫자. 항상.

**콘텐츠 적용:** "이 키워드가 좋을 것 같아" → 서치 콘솔 데이터로 확인.

### 🔄 Iterate — 반복하라
한 번에 맞출 수 없다. 빠르게 만들고, 빠르게 배우고, 빠르게 고친다.

**콘텐츠 적용:** 글 발행 → 2주 후 순위 확인 → 업데이트 → 반복.

### 💀 Kill Fast — 죽일 건 죽여라
안 되는 걸 붙잡고 있으면 기회비용이 죽인다. 피봇 ≠ 실패.

**콘텐츠 적용:** 3개월 해도 트래픽 0인 키워드 → 과감히 버리고 다른 키워드로.

:::tip 🎯 콘텐츠에 적용하면?
주 1개 발행(Ship) → 서치 콘솔(Data) → 업데이트(Iterate) → 안 되는 키워드 버리기(Kill)
:::
