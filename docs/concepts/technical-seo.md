# 테크니컬 SEO 이해하기

> 건물의 인테리어(콘텐츠)보다 먼저, 기초 공사(테크니컬 SEO)가 되어야 합니다.

---

## 테크니컬 SEO란?

**검색엔진(구글 봇)이 내 사이트를 잘 읽고, 좋아할 수 있도록 기술적으로 최적화하는 것.**

쉽게 말하면:

| 비유 | 콘텐츠 SEO | 테크니컬 SEO |
|------|----------|-------------|
| 건물 | 인테리어, 가구 | 기초 공사, 철근 구조 |
| 음식점 | 맛있는 요리 | 위생 등급, 주방 시설 |
| 자동차 | 외관 디자인 | 엔진 성능, 안전 등급 |

> 💡 맛집이라도 위생 등급이 D면 손님이 안 옵니다.\
> 구글도 마찬가지입니다. 콘텐츠가 아무리 좋아도 기술 점수가 낮으면 상위 노출이 안 됩니다.

---

## 구글 봇의 사이트 평가 방식

구글 봇은 전 세계 모든 웹사이트를 돌아다니며 평가합니다.

```
구글 봇이 내 사이트 방문
    ↓
"이 사이트 빠른가?"          → Performance
"모든 사람이 쓸 수 있는가?"   → Accessibility
"웹 표준을 지키는가?"         → Best Practices
"검색엔진이 잘 읽는가?"       → SEO
    ↓
4가지 점수 합산 = Lighthouse Score
    ↓
점수 높을수록 → 검색 결과 상위 노출
```

## 핵심 개념 4가지

### 1. Core Web Vitals (코어 웹 바이탈)

구글이 **가장 중요하게 보는** 3가지 성능 지표입니다.

| 지표 | 무슨 뜻? | 기준 | 쉬운 비유 |
|------|---------|------|----------|
| **LCP** (Largest Contentful Paint) | 가장 큰 요소가 뜨는 시간 | 2.5초 이하 | "문 열었는데 불이 언제 켜지냐" |
| **FID** (First Input Delay) | 클릭했을 때 반응 속도 | 100ms 이하 | "버튼 눌렀는데 언제 반응하냐" |
| **CLS** (Cumulative Layout Shift) | 화면이 갑자기 밀리는 정도 | 0.1 이하 | "읽고 있는데 갑자기 화면이 움직이냐" |

:::warning
**CLS가 가장 흔한 감점 요인입니다.** 이미지에 `width`, `height`를 안 넣으면 발생합니다.\
→ 우리 템플릿은 이걸 자동으로 처리합니다.
:::

### 2. 시맨틱 HTML (Semantic HTML)

구글 봇에게 **"여기가 헤더야, 여기가 본문이야"** 알려주는 구조입니다.

```html
<!-- ❌ 구글 봇: "이게 뭔지 모르겠는데..." -->
<div>제목</div>
<div>내용</div>
<div>하단</div>

<!-- ✅ 구글 봇: "오, 깔끔하게 정리됐네!" -->
<header>제목</header>
<main>
  <article>내용</article>
</main>
<footer>하단</footer>
```

> 💡 직접 짤 필요 없습니다. 우리 템플릿에 이미 적용되어 있습니다.

### 3. 메타 태그 & 구조화 데이터

구글에게 **"이 페이지는 이런 내용이야"** 알려주는 명함입니다.

```html
<!-- 메타 태그: 검색 결과에 보이는 제목/설명 -->
<meta name="title" content="맥북 에어 M3 발열 해결법 5가지">
<meta name="description" content="맥북 에어 M3 발열 문제, 이 5가지로 해결하세요...">

<!-- 구조화 데이터: 구글에 추가 정보 제공 -->
<script type="application/ld+json">
{
  "@type": "BlogPosting",
  "headline": "맥북 에어 M3 발열 해결법 5가지",
  "datePublished": "2026-01-15"
}
</script>
```

**효과:**

| 적용 전 | 적용 후 |
|---------|---------|
| 검색 결과에 제목만 표시 | 날짜, 별점, 이미지까지 표시 (리치 스니펫) |
| 클릭률 3% | 클릭률 8~15% |

> 💡 이것도 직접 짤 필요 없습니다. 우리 템플릿이 자동으로 생성합니다.

### 4. 사이트맵 & Robots.txt

| 파일 | 역할 | 비유 |
|------|------|------|
| `sitemap.xml` | 내 사이트의 전체 지도를 구글에 제출 | 건물 도면을 시청에 제출 |
| `robots.txt` | 구글 봇에게 "여기는 봐, 여기는 보지 마" 안내 | 건물 출입 안내문 |

## 테크니컬 SEO vs 콘텐츠 SEO

| | 테크니컬 SEO | 콘텐츠 SEO |
|---|---|---|
| **비유** | 건물의 기초/구조 | 건물의 인테리어/입주민 |
| **작업** | 사이트 속도, 구조, 접근성 | 키워드 선정, 글쓰기 |
| **빈도** | 처음 한 번 세팅하면 끝 | 지속적으로 글 작성 |
| **담당** | 우리 템플릿이 해결 ✅ | 여러분이 하는 것 |

> 💡 **이 강의의 혁신:**\
> 테크니컬 SEO는 템플릿이 다 해결합니다.\
> 여러분은 콘텐츠(글)에만 집중하면 됩니다.
