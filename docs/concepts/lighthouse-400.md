# Lighthouse 400점 해부

> 구글 봇이 가장 좋아하는 완벽한 구조. 이걸 우리는 0원으로 달성합니다.

---

## Lighthouse란?

**구글이 직접 만든 웹사이트 품질 측정 도구**입니다.

쉽게 말하면, 사이트의 **성적표**예요.\
4가지 과목이 있고, 각각 100점 만점입니다. 합치면 **400점 만점.**

```
┌────────────────────────────────────────────┐
│            LIGHTHOUSE SCORE                │
│                                            │
│  Performance ████████████████████ 100       │
│  (성능 - 사이트가 빠른가?)                   │
│                                            │
│  Accessibility ████████████████████ 100     │
│  (접근성 - 누구나 사용할 수 있는가?)          │
│                                            │
│  Best Practices ████████████████████ 100    │
│  (모범사례 - 웹 표준을 지키는가?)             │
│                                            │
│  SEO ████████████████████ 100              │
│  (검색최적화 - 구글이 잘 읽을 수 있는가?)     │
│                                            │
│  TOTAL: 400 / 400  ✅                      │
└────────────────────────────────────────────┘
```

> 💡 400점에 가까울수록 구글이 좋아합니다.\
> 구글이 좋아하면? **검색 결과 상위에 노출됩니다.**

---

## 각 항목 쉽게 이해하기

### 🟢 Performance (성능) — "이 사이트, 빠른가?"

사이트가 느리면 방문자가 떠납니다.\
구글은 빠른 사이트를 좋아합니다. 이게 **가장 중요한** 항목이에요.

| 측정 항목 | 쉬운 뜻 | 합격 기준 |
|----------|---------|----------|
| FCP | 화면에 뭔가 처음 보이는 시간 | 1.8초 이하 |
| LCP | 가장 큰 내용이 뜨는 시간 | 2.5초 이하 |
| TBT | 화면이 멈추는(버벅이는) 시간 | 0.2초 이하 |
| CLS | 화면이 갑자기 움직이는 정도 | 0.1 이하 |
| Speed Index | 전체적으로 얼마나 빠르게 느껴지는지 | 3.4초 이하 |

**우리 템플릿이 100점을 만드는 방법:**
- 필요한 부분만 불러옴 (전체를 한 번에 안 불러오니까 빠릅니다)
- 이미지를 자동으로 가벼운 형식(WebP)으로 바꿔줌
- 미리 만들어둔 페이지를 보여줌 (매번 새로 만들지 않음)

### 🟣 Accessibility (접근성) — "누구나 쓸 수 있는가?"

"접근성이 왜 중요해요? 시각 장애인분들이 제 블로그를 보나요?"

솔직히 말하면, **구글이 점수에 반영하니까 중요합니다.**\
구글은 접근성 높은 사이트를 좋아합니다. 높을수록 상위 노출이 잘 됩니다.

| 체크 항목 | 쉬운 뜻 |
|----------|---------|
| 이미지 alt 태그 | 이미지에 "이 그림이 뭔지" 설명이 있는가 |
| 색상 대비 | 글씨가 배경과 충분히 구분되는가 (눈이 나쁜 분도 읽을 수 있는가) |
| 키보드 네비게이션 | 마우스 없이 키보드만으로도 사이트를 쓸 수 있는가 |
| ARIA 라벨 | 화면 읽어주는 프로그램이 이해할 수 있는 구조인가 |

> 💡 여러분이 이걸 일일이 신경 쓸 필요 없습니다.\
> **우리 템플릿에 전부 적용되어 있습니다.** 자동이에요.

### 🟡 Best Practices (모범 사례) — "웹 표준을 지키는가?"

인터넷에는 "이렇게 만들어야 한다"는 기준이 있습니다.\
이걸 지키는지 체크합니다.

| 체크 항목 | 쉬운 뜻 | 우리 상태 |
|----------|---------|----------|
| HTTPS | 보안 연결 (주소 앞에 🔒 자물쇠가 있는 것) | ✅ Vercel이 무료로 해줌 |
| 콘솔 에러 | 사이트에 오류가 없는지 | ✅ 템플릿에 없음 |
| 최신 기술 | 구식 코드를 안 쓰는지 | ✅ 템플릿이 최신 |

> 💡 이것도 **전부 자동**입니다. 여러분이 할 것 없어요.

### 🔵 SEO — "구글이 잘 읽을 수 있는가?"

구글 봇이 사이트를 방문했을 때, 내용을 잘 이해할 수 있는지 체크합니다.

| 체크 항목 | 쉬운 뜻 | 우리 상태 |
|----------|---------|----------|
| 메타 태그 | 검색 결과에 보이는 제목과 설명이 있는지 | ✅ 자동 생성 |
| 사이트맵 | 내 사이트의 지도를 구글에 줬는지 | ✅ 자동 생성 |
| 모바일 대응 | 스마트폰에서도 잘 보이는지 | ✅ 자동 적용 |
| H1 태그 | 각 페이지에 제목이 하나만 있는지 | ✅ 자동 관리 |

> 💡 **전부 자동**입니다. 감이 오시죠?\
> 여러분이 코드를 만질 일이 없습니다.

## 400점 vs 250점 — 실제로 얼마나 차이 나나요?

**같은 내용의 글**을 올렸을 때:

| | 250점 사이트 | 400점 사이트 |
|---|---|---|
| 구글에 뜨는 속도 | 3~7일 걸림 | **24시간 이내** |
| 평균 검색 순위 | 15~30위 (2~3페이지) | **3~10위 (1페이지!)** |
| 클릭률 | 1~3% | **5~15%** |
| 월 트래픽 | 500명 | **5,000명** |

> 🔥 **같은 글, 같은 노력인데 10배 차이.**\
> 차이를 만드는 건 **테크니컬 SEO** 점수입니다.\
> 그리고 이걸 **설치만으로 해결하는 게** 이 강의의 핵심입니다.

구글 검색 1페이지에 나오는 것과 2페이지에 나오는 것은 하늘과 땅 차이입니다.\
대부분의 사람들은 **1페이지만 보고 끝**이에요. 2페이지는 거의 안 봅니다.

## 직접 측정해보기

여러분도 아무 사이트의 Lighthouse 점수를 측정할 수 있습니다!

### 방법 1: 크롬 브라우저에서 (컴퓨터)

1. **크롬(Chrome)** 브라우저를 엽니다
2. 측정하고 싶은 사이트에 들어갑니다
3. 키보드에서 **F12** 키를 누릅니다 (Mac은 `Cmd + Option + I`)
4. 위쪽 탭에서 **"Lighthouse"**를 클릭합니다
5. **"Analyze page load"** 버튼을 클릭합니다
6. 30초~1분 기다리면 점수가 나옵니다!

### 방법 2: 온라인 도구 (더 쉬움!)

웹사이트 주소만 넣으면 자동으로 측정해줍니다:

| 도구 | 주소 | 설명 |
|------|------|------|
| **PageSpeed Insights** | [pagespeed.web.dev](https://pagespeed.web.dev) | 구글 공식 도구. 가장 정확 |
| **web.dev** | [web.dev/measure](https://web.dev/measure) | 구글 공식. 더 자세한 분석 |

### 한번 해보세요!

1. [pagespeed.web.dev](https://pagespeed.web.dev)에 접속합니다
2. 네이버 블로그 주소를 하나 넣어보세요
3. 그다음 잘 만든 사이트 주소를 넣어보세요
4. **점수 차이를 눈으로 확인하면** "아, 이래서 400점이 중요하구나" 체감이 됩니다!
