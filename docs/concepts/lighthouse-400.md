## 각 항목 상세

### 🟢 Performance (성능) — 100점

**"이 사이트, 빠른가?"**

| 측정 항목 | 의미 | 합격 기준 |
|----------|------|----------|
| FCP (First Contentful Paint) | 첫 화면이 보이는 시간 | 1.8초 이하 |
| LCP (Largest Contentful Paint) | 메인 콘텐츠 로딩 시간 | 2.5초 이하 |
| TBT (Total Blocking Time) | 화면이 멈추는 시간 | 200ms 이하 |
| CLS (Cumulative Layout Shift) | 화면 떨림 정도 | 0.1 이하 |
| Speed Index | 전체 로딩 체감 속도 | 3.4초 이하 |

**우리 템플릿이 100점을 만드는 방법:**
- Next.js의 자동 코드 분할 (필요한 것만 로딩)
- 이미지 자동 WebP 변환 + lazy loading
- 정적 사이트 생성 (SSG) → 서버 응답 0.1초 미만

### 🟣 Accessibility (접근성) — 100점

**"모든 사람이 쓸 수 있는가?"**

| 체크 항목 | 의미 |
|----------|------|
| 이미지 alt 태그 | 시각 장애인을 위한 이미지 설명 |
| 색상 대비 4.5:1 | 저시력자도 읽을 수 있는 대비 |
| 키보드 네비게이션 | 마우스 없이도 사용 가능 |
| ARIA 라벨 | 스크린 리더가 읽을 수 있는 구조 |

> 💡 "접근성이 왜 중요해요?" → 구글이 점수에 반영하기 때문입니다.\
> 접근성 높은 사이트 = 구글이 좋아하는 사이트 = 상위 노출

**우리 템플릿:** 시맨틱 HTML + ARIA 속성 + 색상 대비 → 자동 적용

### 🟡 Best Practices (모범 사례) — 100점

**"웹 표준을 지키는가?"**

| 체크 항목 | 의미 |
|----------|------|
| HTTPS 사용 | 보안 연결 (Vercel 무료 제공) |
| 콘솔 에러 없음 | JavaScript 오류 0건 |
| 최신 API 사용 | 구식 코드 사용 안 함 |
| 안전한 외부 링크 | `rel="noopener"` 적용 |

**우리 템플릿:** Vercel 배포 시 HTTPS 자동 + Next.js 최신 표준 준수

### 🔵 SEO — 100점

**"검색엔진이 잘 읽을 수 있는가?"**

| 체크 항목 | 의미 |
|----------|------|
| 메타 태그 완비 | title, description 자동 생성 |
| 시맨틱 구조 | header, main, article 태그 |
| sitemap.xml | 구글에 사이트 지도 제출 |
| robots.txt | 크롤링 안내 |
| 모바일 반응형 | 스마트폰 최적화 |
| H1 태그 관리 | 페이지당 1개, 중복 방지 |

**우리 템플릿:** SEO 구조 **강제 시스템** 내장
- 이미지는 무조건 WebP 변환
- H1 태그 중복 방지
- 메타 태그 자동 생성

## 400점 vs 250점: 실제 차이

같은 주제의 글을 올렸을 때:

| | 250점 사이트 | 400점 사이트 |
|---|---|---|
| 구글 인덱싱 속도 | 3~7일 | 24시간 이내 |
| 평균 검색 순위 | 15~30위 (2~3페이지) | 3~10위 (1페이지) |
| 클릭률 (CTR) | 1~3% | 5~15% |
| 월 트래픽 | 500명 | 5,000명 |

> **같은 글, 같은 노력. 10배 차이.**\
> 차이를 만드는 건 테크니컬 SEO입니다.

## 측정하는 법

### 방법 1: Chrome 개발자 도구

```
1. Chrome에서 사이트 열기
2. F12 (Mac: Cmd+Option+I)
3. Lighthouse 탭 클릭
4. 카테고리 4개 체크
5. "Analyze page load" 클릭
6. 30초~1분 대기
```

### 방법 2: 온라인 측정

| 도구 | URL |
|------|-----|
| PageSpeed Insights (구글 공식) | [pagespeed.web.dev](https://pagespeed.web.dev) |
| web.dev | [web.dev/measure](https://web.dev/measure) |
